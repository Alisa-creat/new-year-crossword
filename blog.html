14.01.26
Сегодня я разобралась, из каких имён и на каких уровнях иерархии состоит доменное имя моего сайта:
https://alisa-creat.github.io/new-year-crossword/

Настоящее доменное имя — это alisa-creat.github.io.
Оно состоит из следующих уровней (справа налево):

— . (точка) — корневой домен. Это невидимый, но обязательный уровень всей DNS-системы. В браузерах его не показывают, но технически он есть.

— .io — домен верхнего уровня (TLD). Изначально принадлежит Британской территории в Индийском океане, но сейчас популярен среди разработчиков.

— github — домен второго уровня (SLD). Принадлежит компании GitHub.

— alisa-creat — мой поддомен третьего уровня, созданный автоматически по моему имени пользователя на GitHub.

А вот /new-year-crossword/ — это не часть домена, а путь к конкретному проекту (названию репозитория).

Теперь я понимаю, как устроен интернет — от корня до страницы! 😊

Пример использования модели машинного обучения

Один прогноз (выложить на свой сайт)
new_student = pd.DataFrame({'Часы': [5]})
prediction = regressor.predict(new_student)
print(f"Студент занимался 5 часов")

Несколько прогнозов
# Группа студентов
students = pd.DataFrame({'Часы': [1, 3, 5, 7, 10]})

# Прогноз для всех
predictions = regressor.predict(students)

Этот код делает прогноз балла по количеству часов учёбы. Модель уже обучена — осталось только подать новые данные и получить ответ. Как будто у тебя есть «умный помощник», который знает: чем больше учишься — тем выше результат! 😊
